## Item 1: Nacht-Stopp Automatisierung

### Kontext

v0.4.0 hat ein manuelles `playbooks/nacht-stopp.yml` das nur den Kill Switch
setzt (`jobs_enabled=false`). Jetzt wird das automatisiert via systemd Timer auf
dem PROD-Server.

### Design-Entscheidung: Nur Docker stoppen, nicht den Server

Der Timer läuft auf dem PROD-Server selbst. Deshalb kann der Server sich nicht
selbst via UpCloud API stoppen — nach dem Stop würde der Start-Timer nicht mehr
feuern. Stattdessen:

- **22:00**: Kill Switch → Docker Stop (Server bleibt an, ~0 Ressourcen)
- **08:00**: Docker Start → Kill Switch aufheben → Smoke-Test

Das spart Docker-Overhead in der Nacht. Für vollen Server-Stop (Kostenoptimierung)
kann später UpCloud Scheduled Actions oder ein externer Trigger ergänzt werden.

### Was existiert (v0.4.0)

- `playbooks/nacht-stopp.yml` — setzt `jobs_enabled=false` via Ansible
- `runtime_control` Tabelle: `env_name='prod'`, `mode`, `traffic_enabled`, `jobs_enabled`
- DB-Zugriff auf PROD: `docker exec factory-postgres psql -U forgejo -d forgejo`

### Was gebaut werden muss

#### 1.1 Stop-Script

**Datei:** `roles/nightstop/templates/nightstop.sh.j2`

```bash
#!/bin/bash
# Fährt Code-Fabrik Nacht-Pause ein (Docker stoppen, Kill Switch)
set -euo pipefail

PSQL="docker exec factory-postgres psql -U forgejo -d forgejo -tAq"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [nightstop] $*" | systemd-cat -t codefabrik -p info; }
err() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [nightstop] ERROR: $*" | systemd-cat -t codefabrik -p err; }

# 1. Prüfe ob Nacht-Stopp manuell deaktiviert wurde (reason='nightstop-disabled')
REASON=$($PSQL -c "SELECT reason FROM runtime_control WHERE env_name='prod'" 2>/dev/null | tr -d '[:space:]')
if [ "$REASON" = "nightstop-disabled" ]; then
  log "Nacht-Stopp manuell deaktiviert (reason=nightstop-disabled). Überspringe."
  exit 0
fi

# 2. Kill Switch setzen
log "Setze Kill Switch (jobs_enabled=false, mode=MAINTENANCE)"
$PSQL -c "
  UPDATE runtime_control SET
    jobs_enabled=false,
    mode='MAINTENANCE',
    reason='Nacht-Stopp',
    updated_by='nightstop',
    updated_at=NOW()
  WHERE env_name='prod';" || err "Kill Switch setzen fehlgeschlagen"

# 3. Warte auf laufende Poller-Instanz (max 5 Min)
log "Warte auf laufende Poller..."
for i in $(seq 1 30); do
  if [ ! -f /tmp/openclaw-poller.lock ]; then
    log "Kein Poller aktiv."
    break
  fi
  OLD_PID=$(cat /tmp/openclaw-poller.lock 2>/dev/null || echo "0")
  if ! kill -0 "$OLD_PID" 2>/dev/null; then
    log "Poller-Lock veraltet (PID $OLD_PID tot)."
    break
  fi
  if [ "$i" = "30" ]; then
    err "Timeout: Poller (PID $OLD_PID) nach 5 Min noch aktiv. Stoppe trotzdem."
  fi
  sleep 10
done

# 4. Docker Stack stoppen (außer Postgres — brauchen wir für den Morning-Check)
log "Stoppe Docker Services..."
cd /opt/codefabrik
docker compose stop forgejo openclaw caddy gateway || err "Docker compose stop fehlgeschlagen"

log "Nacht-Stopp abgeschlossen."
```

#### 1.2 Start-Script

**Datei:** `roles/nightstop/templates/nightstart.sh.j2`

```bash
#!/bin/bash
# Morgen-Start: Docker hoch, Kill Switch aufheben, Smoke-Test
set -euo pipefail

PSQL="docker exec factory-postgres psql -U forgejo -d forgejo -tAq"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [nightstart] $*" | systemd-cat -t codefabrik -p info; }
err() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [nightstart] ERROR: $*" | systemd-cat -t codefabrik -p err; }

# 1. Docker Stack starten
log "Starte Docker Services..."
cd /opt/codefabrik
docker compose up -d || { err "Docker compose up fehlgeschlagen"; exit 1; }

# 2. Warte auf Services (max 2 Min)
log "Warte auf Services..."
for i in $(seq 1 24); do
  if docker exec factory-postgres pg_isready -U forgejo -q 2>/dev/null; then
    log "PostgreSQL bereit."
    break
  fi
  if [ "$i" = "24" ]; then
    err "Timeout: PostgreSQL nach 2 Min nicht bereit"
    exit 1
  fi
  sleep 5
done

# 3. Warte auf Forgejo
for i in $(seq 1 12); do
  HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:3000/api/v1/version" 2>/dev/null || echo "0")
  if [ "$HTTP_CODE" = "200" ]; then
    log "Forgejo bereit."
    break
  fi
  if [ "$i" = "12" ]; then
    err "Timeout: Forgejo nach 1 Min nicht bereit"
    exit 1
  fi
  sleep 5
done

# 4. Kill Switch aufheben
log "Setze LIVE + jobs_enabled=true"
$PSQL -c "
  UPDATE runtime_control SET
    mode='LIVE',
    traffic_enabled=true,
    jobs_enabled=true,
    reason='',
    updated_by='nightstart',
    updated_at=NOW()
  WHERE env_name='prod';" || err "Kill Switch aufheben fehlgeschlagen"

# 5. Gateway Smoke-Test
for i in $(seq 1 6); do
  HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:3100/ready" 2>/dev/null || echo "0")
  if [ "$HTTP_CODE" = "200" ]; then
    log "Smoke-Test bestanden: /ready = 200"
    break
  fi
  if [ "$i" = "6" ]; then
    err "Smoke-Test fehlgeschlagen: /ready = $HTTP_CODE nach 30s"
    exit 1
  fi
  sleep 5
done

log "Morgen-Start abgeschlossen."
```

## Item 2: Poller Retry-Logik

### Kontext

Der Poller (380 Zeilen Bash) hat keine Retry-Logik. Bei transienten Fehlern
(Claude API Timeout, Netzwerkprobleme, Docker-Hänger) bricht er sofort ab.
Exponential Backoff mit max 3 Retries für die kritische Phase 3 (OpenClaw).

### Ist-Stand: openclaw-poller.sh.j2

Phase 3 (Zeile 311-318) führt Claude Code aus:
```bash
if docker exec factory-openclaw su junior -c "bash $SCRIPT_FILE" 2>&1 | \
  tee -a "$LOGFILE"; then
  update_poller_state "$story_id" "openclaw-done" "$BRANCH_NAME"
else
  update_poller_state "$story_id" "openclaw-error" "$BRANCH_NAME"
fi
```

Bei Fehler wird sofort `openclaw-error` gesetzt. Kein Retry, kein Event-Log.

### Was gebaut werden muss

#### 2.1 Neue DB-Tabelle: poller_events

#### 2.2 Retry-Funktion im Poller

**Am Anfang von `openclaw-poller.sh.j2` nach den bestehenden Hilfsfunktionen:**

```bash
# ─── Retry mit Exponential Backoff ───────────────────────────────────────────
# Nutzung: retry_with_backoff MAX_RETRIES "Beschreibung" command arg1 arg2
retry_with_backoff() {
  local max_retries=$1
  local description=$2
  shift 2

  local attempt=0
  local wait_seconds=10  # 10s → 30s → 90s

  while [ $attempt -lt $max_retries ]; do
    attempt=$((attempt + 1))
    log "[$description] Versuch $attempt/$max_retries"

    if "$@"; then
      log "[$description] Erfolg nach Versuch $attempt"
      return 0
    fi

    if [ $attempt -lt $max_retries ]; then
      log "[$description] Fehlgeschlagen. Warte ${wait_seconds}s..."
      sleep $wait_seconds
      wait_seconds=$((wait_seconds * 3))
    fi
  done

  err "[$description] Fehlgeschlagen nach $max_retries Versuchen"
  return 1
}

log_poller_event() {
  local object_id="$1"
  local event="$2"
  local details="${3:-}"
  docker exec factory-postgres psql -U forgejo -d forgejo -tAq -c "
    INSERT INTO poller_events (object_id, event, details)
    VALUES ('$object_id', '$event', '$(echo "$details" | head -c 500)');" 2>/dev/null || true
}
```

#### 2.3 Phase 3 mit Retry wrappen

**Ersetze Zeile 311-318 in `openclaw-poller.sh.j2`:**

```bash
  run_openclaw_task() {
    docker exec factory-openclaw su junior -c "bash $SCRIPT_FILE" 2>&1 | \
      tee -a "$LOGFILE"
  }

  if retry_with_backoff 3 "openclaw:$story_id" run_openclaw_task; then
    update_poller_state "$story_id" "openclaw-done" "$BRANCH_NAME"
    log_poller_event "$story_id" "openclaw-done" "Implementierung abgeschlossen"
    log "openclaw: $story_id Implementierung abgeschlossen"
  else
    update_poller_state "$story_id" "openclaw-error" "$BRANCH_NAME"
    log_poller_event "$story_id" "retry_exhausted" "Fehlgeschlagen nach 3 Versuchen"
    log "openclaw: $story_id Fehler nach 3 Versuchen"
  fi
```

## Item 3: Basic Metriken

### Kontext

Pipeline-Durchlaufzeiten werden nicht gemessen. Timestamps an kritischen
Stellen erfassen. Alle Daten landen in einer DB-Tabelle, die der Poller
progressiv füllt.

### Was gebaut werden muss

#### 3.1 DB-Schema

**In `roles/postgres/tasks/main.yml` ergänzen:**

```yaml
- name: "pipeline_metrics Tabelle erstellen"
  ansible.builtin.shell: |
    docker exec factory-postgres psql -U forgejo -d forgejo -tAq -c "
    CREATE TABLE IF NOT EXISTS pipeline_metrics (
        id                      SERIAL PRIMARY KEY,
        object_id               TEXT NOT NULL UNIQUE,
        product                 TEXT,
        dispatched_at           TIMESTAMPTZ,
        in_progress_at          TIMESTAMPTZ,
        pr_created_at           TIMESTAMPTZ,
        merged_at               TIMESTAMPTZ,
        blocked_at              TIMESTAMPTZ,
        total_duration_minutes  INTEGER,
        dev_duration_minutes    INTEGER,
        retry_count             INTEGER DEFAULT 0,
        blocked_reason          TEXT
    );"

- name: "pipeline_metrics Dauer-Trigger"
  ansible.builtin.shell: |
    docker exec factory-postgres psql -U forgejo -d forgejo -tAq -c "
    CREATE OR REPLACE FUNCTION update_pipeline_durations()
    RETURNS TRIGGER AS \$\$
    BEGIN
      IF NEW.merged_at IS NOT NULL AND NEW.dispatched_at IS NOT NULL THEN
        NEW.total_duration_minutes := EXTRACT(EPOCH FROM (NEW.merged_at - NEW.dispatched_at)) / 60;
      END IF;
      IF NEW.pr_created_at IS NOT NULL AND NEW.in_progress_at IS NOT NULL THEN
        NEW.dev_duration_minutes := EXTRACT(EPOCH FROM (NEW.pr_created_at - NEW.in_progress_at)) / 60;
      END IF;
      RETURN NEW;
    END;
    \$\$ LANGUAGE plpgsql;

    DROP TRIGGER IF EXISTS trg_pipeline_durations ON pipeline_metrics;
    CREATE TRIGGER trg_pipeline_durations
      BEFORE INSERT OR UPDATE ON pipeline_metrics
      FOR EACH ROW EXECUTE FUNCTION update_pipeline_durations();"
```

#### 3.2 Hilfsfunktion im Poller

**In `openclaw-poller.sh.j2` nach den bestehenden Hilfsfunktionen:**

```bash
record_metric() {
  local object_id="$1"
  local field="$2"
  local value="${3:-NOW()}"
  docker exec factory-postgres psql -U forgejo -d forgejo -tAq -c "
    INSERT INTO pipeline_metrics (object_id, $field)
    VALUES ('$object_id', $value)
    ON CONFLICT (object_id)
    DO UPDATE SET $field = $value;" 2>/dev/null || true
}
```

#### 3.3 Integration in die 4 Phasen

**Phase 1 (dispatch) — nach `update_poller_state`:**
```bash
  record_metric "$story_id" "dispatched_at"
  record_metric "$story_id" "product" "'$produkt'"
```

**Phase 2 (bridge) — nach Branch-Erstellung:**
```bash
  record_metric "$story_id" "in_progress_at"
```

**Phase 3 (openclaw) — bei Retry-Erschöpfung:**
```bash
  # Im Fehlerfall:
  record_metric "$story_id" "blocked_at"
  record_metric "$story_id" "blocked_reason" "'retry_exhausted'"
  record_metric "$story_id" "retry_count" "3"
```

**Phase 4 (pr-check) — nach PR-Erstellung:**
```bash
  record_metric "$story_id" "pr_created_at"
```

#### 3.4 Akzeptanzkriterien

- [ ] `pipeline_metrics` Tabelle existiert mit Trigger

- [ ] Trigger berechnet `total_duration_minutes` und `dev_duration_minutes`
- [ ] Blockierte Stories werden mit `blocked_at` + `blocked_reason` dokumentiert

## Item 4: Gateway /metrics Endpoint

### Kontext

Pipeline-Statistiken sollen über den Gateway abfragbar sein. Der Gateway
verwendet raw Node.js `http.createServer` mit `pg.Client` (kein Express,
kein Connection Pool).

### Ist-Stand: server.js

```javascript
const http = require('http');
const { Client } = require('pg');

const server = http.createServer(async (req, res) => {
  // /health → 200 (status, version, uptime)
  // /ready  → 200/503 (runtime_control check via pg.Client)
  // *       → 404
});
```

### Was gebaut werden muss

**Neuer Block in `roles/gateway/tasks/main.yml` → server.js:**

```javascript
    } else if (req.url === '/metrics') {
      const client = new Client({
        host: 'postgres',
        port: 5432,
        database: 'forgejo',
        user: 'forgejo',
        password: process.env.POSTGRES_PASSWORD,
        connectionTimeoutMillis: 5000
      });
      try {
        await client.connect();
        const { rows } = await client.query(`
          SELECT
            COUNT(*) FILTER (WHERE pr_created_at IS NOT NULL AND blocked_at IS NULL) as completed,
            COUNT(*) FILTER (WHERE blocked_at IS NOT NULL) as blocked,
            COUNT(*) FILTER (WHERE pr_created_at IS NULL AND blocked_at IS NULL) as in_progress,
            ROUND(AVG(total_duration_minutes) FILTER (WHERE total_duration_minutes IS NOT NULL)) as avg_duration_min,
            ROUND(AVG(dev_duration_minutes) FILTER (WHERE dev_duration_minutes IS NOT NULL)) as avg_dev_min,
            MAX(pr_created_at) as last_pr
          FROM pipeline_metrics
        `);
        await client.end();
        res.writeHead(200);
        res.end(JSON.stringify({
          pipeline: rows[0],
          version: '{{ factory_version }}',
          timestamp: new Date().toISOString()
        }));
      } catch (e) {
        try { await client.end(); } catch (_) {}
        res.writeHead(500);
        res.end(JSON.stringify({ error: 'metrics unavailable', reason: e.message }));
      }
```

### Akzeptanzkriterien

- [ ] `GET /metrics` gibt Pipeline-Statistiken als JSON zurück
